阿里云对象存储定时备份 gogs 和 kanboard

Posted on 2019-05-29 | Edited on 2019-05-31 | Comments: | Views:

背景

由于国内访问 github 速度比较慢,而且为了方便放一些代码,所以在阿里云服务器上搭建了一个 gogs 版本库.
另外还搭建了一个 kanboard .这是一个看板应用,我用来做项目进度管理和日常知识记录.
gogs 和 kanboard 都是我自建的服务

gogs 备份

具体搭建 gogs 的过程先不讲了,可以参考官方文档.这里主要说一下如何备份 gogs 的信息
在 gogs 的根目录下执行 ./gogs -h 可以看到提供了 backup 参数

通过 执行 ./gogs backup -h 来查看具体的细节和可以配置的参数

可以通过 传递 target 和 archive-name 参数来控制生成的文件的路径和文件名,得到完整路径后就可以通过 ossutil64 命令上传到阿里云 oss 对象存储中

坑1:
如果执行在非 gogs 根目录下执行 backup 命令时,报错误信息为
[FATAL] Fail to dump database: fail to dump table ‘User’: no such table: user .这是因为在 gogs 的配置文件 custom/conf/app.ini 中配置的 sqlite3 文件路径为相对路径,导出时找不到对应的文件. 需要在改为绝对路径即可

kanboard 备份

kanboard 我是通过 docker 搭建的,非常方便.搭建的命令中将 kanboard 的数据库文件目录挂载到宿主机上,所以直接就能拿到需要备份的文件

挂载了 /var/www/app/data 目录在宿主机的 /root/kanboard/kanboard_data 目录,目录下的 db.sqlite 文件即为 kanboard 的数据库文件

配置 ossutil64

登录阿里云 oss 控制台,购买资源包,新建 bucket并创建 accessKey
接着根据文档完成下列步骤

编写备份文件脚本

kanboard_backup.sh

gogs_backup.sh

添加到 crontab 定时任务

后记

我定时任务配置的是每天一保存,时间久了会造成 oss 存储空间中包含太多的文件,后续可以编写脚本定时删除过时的备份文件.其实除了使用脚本和命令行软件 ossutil64,阿里云对象存储也提供了 java sdk 来进行文件上传下载等操作
